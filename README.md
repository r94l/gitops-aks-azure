# Azure AKS Deployment with Terraform, Helm & ArgoCD 🚀

[![Terraform](https://img.shields.io/badge/Terraform-1.6.0-blue)](https://www.terraform.io/) [![Azure CLI](https://img.shields.io/badge/Azure%20CLI-latest-green)](https://docs.microsoft.com/en-us/cli/azure/) [![Kubernetes](https://img.shields.io/badge/Kubernetes-1.28-blueviolet)](https://kubernetes.io/) [![Helm](https://img.shields.io/badge/Helm-3.11-orange)](https://helm.sh/)

This project automates the deployment of a production-ready Azure Kubernetes Service (AKS) cluster using Terraform, Helm, and ArgoCD, demonstrating modern DevOps practices including infrastructure-as-code (IaC), continuous deployment, and GitOps workflows. It is designed as a portfolio project for DevOps engineers and recruiters to showcase expertise in Azure cloud, Kubernetes, and automation pipelines. The project provisions the required Azure resources, sets up an AKS cluster, installs Helm-managed applications, and configures ArgoCD for GitOps-based deployment, providing a complete end-to-end DevOps experience.

The technologies used include Terraform for infrastructure-as-code, Azure Kubernetes Service for managed Kubernetes, Helm for package management, ArgoCD for continuous deployment, Azure CLI and kubectl for cluster management, and Git for version control. Prerequisites include an Azure subscription, Terraform >= 1.6, Azure CLI >= 2.50, kubectl >= 1.28, Helm >= 3.11, ArgoCD CLI (optional), and Git.

To deploy the project, first clone the repository using `git clone https://github.com/<your-username>/<your-repo>.git` and navigate into it. Initialize Terraform with `terraform init` inside the Terraform directory, then run `terraform plan` and `terraform apply -auto-approve` to provision the necessary resources including a resource group, AKS cluster, service principal, managed identity, and networking. Once the infrastructure is ready, configure kubectl to connect to the cluster using `az aks get-credentials --resource-group <RG_NAME> --name <AKS_CLUSTER_NAME>` and verify the nodes with `kubectl get nodes`. ArgoCD can then be installed by creating its namespace and applying its manifests using `kubectl create namespace argocd` followed by `kubectl apply -n argocd -f https://raw.githubusercontent.com/argoproj/argo-cd/stable/manifests/install.yaml`. Helm applications can be deployed using `helm repo add stable https://charts.helm.sh/stable` and `helm install my-app stable/<chart-name>`. The ArgoCD dashboard can be accessed locally with `kubectl port-forward svc/argocd-server -n argocd 8080:443` and navigating to `https://localhost:8080` with username `admin` and the initial password retrieved via `kubectl get secret argocd-initial-admin-secret -n argocd -o jsonpath="{.data.password}" | base64 -d`.

This project demonstrates several key DevOps practices: fully automated AKS provisioning using Terraform, GitOps deployment with ArgoCD for continuous delivery, Helm-based management of Kubernetes applications, and the integration of IaC, CI/CD, and cloud-native tools. It serves as a comprehensive portfolio-ready project for DevOps recruiters and professionals to showcase expertise. When finished, resources can be safely removed to avoid unnecessary Azure costs using `terraform destroy -auto-approve`.
